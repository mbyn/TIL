### Context

사용자는 n8n을 활용하여 첫 번째 워크플로우 자동화를 구축했다. 이 워크플로우는 Discord 메시지를 트리거로 사용하여 LLM 에이전트를 실행하고, 에이전트가 생성한 기술 문서(TIL)를 GitHub에 푸시하며, README.md 파일 업데이트 및 완료 알림 메시지 발신까지의 전 과정을 자동화하는 것을 목표로 한다.

### Core

n8n 워크플로우는 다음과 같은 단계로 구성된다.

*   **Discord 메시지 수신 (Discord Trigger)**
    *   Discord 채널에서 특정 메시지가 수신되면 워크플로우를 시작한다.
    *   수신된 메시지의 내용을 다음 노드로 전달한다.

    ```markdown
    - Discord Trigger 노드 설정:
        - Discord 봇 토큰 및 채널 지정.
        - 특정 키워드 또는 모든 메시지를 트리거 조건으로 설정.
    ```

*   **에이전트 실행 및 TIL 변환 (Code / HTTP Request)**
    *   수신된 Discord 메시지를 기반으로 LLM 에이전트를 호출하여 TIL 형식의 마크다운 문서를 생성한다.
    *   이 단계는 n8n의 `Code` 노드 또는 외부 에이전트 API를 호출하는 `HTTP Request` 노드를 통해 구현될 수 있다.

    ```python
    # 예시: Code 노드를 사용하여 TIL 콘텐츠 생성 (가상의 로직)
    # n8n의 'Code' 노드는 JavaScript 또는 Python 스크립트를 실행할 수 있습니다.
    item = items[0].json;
    const discordMessage = item.text; // Discord 메시지 내용
    let tilContent = '';

    // LLM 에이전트 호출 또는 로컬 로직으로 TIL 생성
    // 예: tilContent = callLLMAgent(discordMessage);
    tilContent = `
    ### Context
    ${discordMessage} 내용을 기반으로 생성된 TIL입니다.

    ### Core
    \`\`\`python
    print("TIL 생성 로직 예시")
    \`\`\`

    ### Insight
    생성된 TIL에 대한 인사이트입니다.
    `;

    return [{
        json: {
            filename: "2026-03-02-생성된-TIL-제목.md", // 동적으로 생성된 파일명
            content: tilContent
        }
    }];
    ```

*   **GitHub에 TIL 푸시 (GitHub Node)**
    *   에이전트가 생성한 TIL 마크다운 파일을 GitHub 리포지토리에 푸시한다.
    *   \`Create File\` 또는 \`Update File\` 작업을 사용하여 지정된 디렉토리(\`n8n\`, \`python\`, \`ai-agent\` 중 하나)에 파일을 저장한다.

    ```markdown
    - GitHub 노드 설정:
        - 인증: GitHub OAuth 또는 Personal Access Token (PAT)
        - 작업: 'Create File'
        - 리포지토리: 대상 GitHub 리포지토리
        - 파일 경로: \`{{$json.folder}}/{{$json.filename}}\` (이전 노드에서 생성된 파일명 및 폴더명 사용)
        - 파일 콘텐츠: \`{{$json.content}}\` (이전 노드에서 생성된 TIL 내용)
        - 커밋 메시지: "TIL: {{$json.filename}} 추가"
    ```

*   **README.md 인덱스 업데이트 (GitHub Node & Code)**
    *   GitHub 리포지토리의 \`README.md\` 파일을 읽어온다.
    *   새로 추가된 TIL 문서의 링크를 \`README.md\` 파일의 목록에 추가한다.
    *   업데이트된 \`README.md\` 파일을 다시 GitHub에 푸시한다.

    ```markdown
    - GitHub 노드 (Read File):
        - 작업: 'Get Content of File'
        - 파일 경로: 'README.md'

    - Code 노드 (README.md 업데이트 로직):
        - 기존 README.md 내용에 새로 추가된 TIL 링크(\`* [TIL 제목](폴더/파일명.md)\`)를 추가하는 로직 구현.

    - GitHub 노드 (Update File):
        - 작업: 'Update File'
        - 파일 경로: 'README.md'
        - 파일 콘텐츠: 이전 Code 노드에서 수정된 README.md 내용
        - 커밋 메시지: "README.md: TIL 인덱스 업데이트"
    ```

*   **완료 메시지 발신 (Discord / Slack 등)**
    *   모든 작업이 성공적으로 완료되었음을 Discord 채널에 알리는 메시지를 발신한다.

    ```markdown
    - Discord 노드 (Send Message):
        - 채널: 완료 알림을 보낼 Discord 채널
        - 메시지: "TIL 문서 생성 및 GitHub 푸시가 완료되었습니다: [링크]"
    ```

### Insight

n8n을 활용한 워크플로우 자동화를 처음 구현하는 과정에서 가장 중요하다고 느낀 점은 **각 노드의 입력(Input)과 출력(Output)을 정확히 이해하는 것**이었다. n8n 노드는 데이터를 추출(Extract), 변환(Transform), 적재(Load)하는 ETL(Extract, Transform, Load) 도구의 역할을 수행하며, 한 노드의 출력이 다음 노드의 입력으로 매끄럽게 연결되는 방식을 파악하는 것이 전체 워크플로우 설계의 핵심이다.

예를 들어, \`Discord Trigger\` 노드에서 수신된 메시지 정보가 다음 \`Code\` 노드의 입력으로 어떻게 전달되는지, 그리고 \`Code\` 노드에서 생성된 파일명과 콘텐츠가 \`GitHub\` 노드의 파일 생성 인자로 어떻게 매핑되는지를 명확히 인지해야 한다. 이러한 이해를 바탕으로 워크플로우를 구성하면 복잡한 자동화도 효율적으로 구현할 수 있다.
