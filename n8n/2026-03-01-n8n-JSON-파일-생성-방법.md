### Context
n8n은 워크플로우 자동화 도구로, 다양한 시스템 간의 데이터 흐름을 연결하고 자동화하는 데 강점을 가집니다. 기술 문서나 기록(예: TIL, Today I Learned)을 특정 구조의 JSON 파일로 생성해야 할 때, n8n을 활용하면 수동 작업을 줄이고 일관된 형식으로 데이터를 생성할 수 있습니다. 이는 특히 구조화된 데이터를 요구하는 시스템이나 API에 데이터를 전송할 때 매우 유용합니다.

### Core
n8n을 사용하여 TIL 형식의 JSON 파일을 생성하는 워크플로우는 다음과 같은 주요 노드들로 구성될 수 있습니다.

1.  **Start 노드**: 워크플로우의 시작점입니다. 수동 실행, 스케줄러, 웹훅 등 다양한 방식으로 워크플로우를 시작할 수 있습니다.
2.  **Set 노드**: JSON 파일에 포함될 데이터를 정의합니다. 예를 들어, TIL의 'folder', 'filename', 'content', 'summary'와 같은 필드를 여기서 설정할 수 있습니다.
    ```json
    {
      "folder": "ai-agent",
      "filename": "2026-03-01-n8n-JSON-파일-생성-방법.md",
      "content": "이것은 n8n을 사용하여 생성된 TIL 문서의 본문입니다.",
      "summary": "n8n으로 JSON 파일 생성 워크플로우를 구성합니다."
    }
    ```
3.  **Function 노드**: Set 노드에서 정의한 데이터를 실제 JSON 문자열로 변환합니다. `JSON.stringify()` 메서드를 사용하여 JavaScript 객체를 JSON 형식의 문자열로 직렬화할 수 있습니다.
    ```javascript
    const item = items[0].json;

    // JSON.stringify()를 사용하여 객체를 JSON 문자열로 변환합니다.
    return [{
      json: {
        fileName: item.filename,
        fileContent: JSON.stringify(item, null, 2) // 가독성을 위해 2칸 들여쓰기
      }
    }];
    ```
4.  **Write Binary File 노드**: Function 노드에서 생성된 JSON 문자열을 로컬 파일 시스템에 `.json` 확장자를 가진 파일로 저장합니다. 이 노드에서 파일 경로와 파일 이름을 설정할 수 있습니다. 예를 들어, `fileName` 필드를 파일 이름으로, `fileContent` 필드를 파일 내용으로 매핑할 수 있습니다.

### Insight
n8n을 이용한 JSON 파일 생성은 데이터의 구조적 일관성을 보장하고 수동 파일 생성에 드는 시간을 절약합니다. 특히 `Function` 노드에서 `JSON.stringify()`를 활용하면 복잡한 객체도 손쉽게 JSON 형식으로 변환할 수 있어, 다양한 형태의 데이터를 유연하게 처리할 수 있습니다. 이 과정에서 파일명 생성 규칙, 디렉토리 분류 등 정해진 규칙을 n8n 워크플로우 내에서 구현함으로써 자동화된 문서 관리 시스템을 구축하는 데 기여할 수 있습니다. 또한, `Write Binary File` 노드 대신 `HTTP Request` 노드를 사용하여 생성된 JSON 데이터를 외부 API로 전송하는 방식으로 확장할 수도 있습니다.
