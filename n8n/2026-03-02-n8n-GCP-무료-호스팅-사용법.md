### Context
n8n 워크플로우를 영구적으로 무료로 운영하기 위한 방안으로 Google Cloud Platform(GCP)의 프리 티어(Free Tier) 서비스를 활용하는 방법을 모색합니다. 이를 통해 초기 비용 부담 없이 n8n을 구축하고 운영하는 것을 목표로 합니다.

### Core
*   **GCP 프리 티어**: GCP는 다양한 서비스에 대해 일정 사용량까지 무료로 제공하는 프리 티어를 운영합니다. n8n 운영에 필요한 컴퓨팅 리소스(Compute Engine), 스토리지, 네트워크 트래픽 등이 프리 티어 범위 내에서 사용 가능한지 확인해야 합니다.
*   **Compute Engine 활용**: 프리 티어로 제공되는 e2-micro 인스턴스 또는 유사한 사양의 VM을 사용하여 n8n을 호스팅합니다.
    *   운영체제: Ubuntu, Debian 등 경량 리눅스 배포판 권장
    *   n8n 설치: Node.js를 설치하고 npm을 통해 n8n을 설치합니다. (예: `npm install -g n8n`)
    *   서비스 관리: PM2와 같은 프로세스 관리 도구를 사용하여 n8n을 백그라운드에서 실행하고 자동 재시작 설정을 합니다.
*   **데이터베이스**: n8n은 기본적으로 SQLite를 사용하지만, 안정적인 운영을 위해 PostgreSQL이나 MySQL 사용을 고려할 수 있습니다. GCP의 Cloud SQL 프리 티어 활용 가능 여부를 확인하거나, VM 내부에 직접 설치하여 운영합니다.
*   **도메인 및 HTTPS**: GCP 프리 티어에서는 자체 도메인 연결 및 HTTPS 설정이 제한적일 수 있습니다. 초기에는 GCP에서 제공하는 임시 IP 주소로 접속하거나, Cloudflare와 같은 외부 서비스를 무료로 연동하여 도메인 및 SSL/TLS 설정을 진행합니다.
*   **프리 티어 제한**: 각 GCP 서비스별 프리 티어 사용량을 초과하지 않도록 n8n 워크플로우의 실행 빈도와 복잡도를 조절해야 합니다. 특히 CPU, 메모리, 네트워크 아웃바운드 트래픽 사용량을 주기적으로 모니터링합니다.

```bash
# n8n 설치 예시
curl -fsSL https://raw.githubusercontent.com/n8n-io/n8n/master/scripts/install.sh | bash

# PM2를 이용한 n8n 실행 및 관리
# n8n 설치 후, n8n 실행 경로 확인 (예: ~/.n8n/)
cd ~/.n8n
npm run start:prod
# pm2 start npm --name 'n8n' -- run start:prod
# pm2 save
# pm2 list
# pm2 logs n8n
```

### Insight
GCP 프리 티어를 활용한 n8n 무료 호스팅은 초기 비용 없이 n8n을 경험하고 소규모 자동화 프로젝트를 실행하는 데 매우 유용합니다. e2-micro 인스턴스는 n8n의 기본 기능 및 경미한 워크플로우 실행에는 충분하지만, 복잡하거나 리소스 집약적인 워크플로우를 다수 운영할 경우 프리 티어 제한을 초과할 가능성이 높습니다. 따라서 워크플로우의 효율적인 설계와 리소스 사용량 모니터링이 필수적입니다. 또한, 안정적인 운영을 위해서는 데이터베이스 관리 및 HTTPS 설정 등 추가적인 고려 사항이 필요하며, 이 부분은 GCP 프리 티어만으로는 한계가 있을 수 있어 외부 서비스 연동이 필요할 수 있습니다. 장기적으로 안정적이고 확장 가능한 운영을 원한다면 유료 플랜으로의 전환을 고려해야 합니다.
